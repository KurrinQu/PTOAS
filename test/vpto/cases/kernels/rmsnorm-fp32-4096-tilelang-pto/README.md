# RMSNorm FP32 4096 TileLang PTO Case

This case is generated from TileLang PTO codegen for the RMSNorm shape
`batch=4096, hidden=4096`.

It is intended for manual VPTO/PTO IR experiments, especially for estimating
SIMT fragment persistent/register-residency optimization impact. The expected
workflow is to edit `kernel.pto`, rebuild, and rerun the standalone host
launcher.

## Files

- `kernel.pto`: PTO IR generated by PTODSL from TileLang.
- `kernel.ptodsl.py`: PTODSL source kept for reference.
- `launch.cpp`: TileLang-generated host/device launch wrapper. It exposes
  `extern "C" int call(void *X, void *Y, void *W, void *RSTD, float eps, void *stream)`.
- `main.cpp`: standalone ACL host program. It initializes FP32 input data,
  launches the rebuilt kernel through `call`, and validates RMSNorm output.
- `build.sh`: local build/run helper.

Generated artifacts such as `build/`, `*.o`, `*.so`, and `__pycache__/` are
intentionally not tracked in this case directory.

## Build

```bash
cd /home/qukelin/projects/PTOAS/test/vpto/cases/kernels/rmsnorm-fp32-4096-tilelang-pto
./build.sh build
```

The script rebuilds:

- `build/kernel.fatobj.o` from `kernel.pto` with `ptoas --pto-backend=vpto`
- `build/launch.o` from `launch.cpp` with `bisheng`
- `build/lib_kernel.so`
- `build/rmsnorm_pto_main`

## Run

```bash
./build.sh run
```

or run an already built binary:

```bash
./build.sh run-only
```

Useful environment variables:

- `CANN_HOME`: CANN install path. Default:
  `/home/qukelin/tools/CANN_9.1/cann-9.1.T530`
- `PTOAS_ROOT`: PTOAS source root. Default:
  `/home/qukelin/projects/PTOAS`
- `PTOAS_BIN`: ptoas binary path. Default:
  `${PTOAS_ROOT}/build/tools/ptoas/ptoas`
- `ACL_DEVICE_ID`: Ascend device id. Default: `0`
- `ASCEND_RT_VISIBLE_DEVICES`: visible Ascend devices. Default: `0`
- `WARMUP_ITERS`: warmup launch count. Default: `0`
- `RUN_ITERS`: measured launch count. Default: `1`
- `SKIP_VALIDATION=1`: skip output validation and only launch the kernel.

Example:

```bash
RUN_ITERS=100 SKIP_VALIDATION=1 ./build.sh run
```

## Manual IR Experiment Flow

1. Edit `kernel.pto`. For SIMT fragment persistent/register-residency
   experiments, use `pto.keep` / `pto.resume` to preserve and restore
   per-workitem scalar values across adjacent SIMT entry calls. See
   `/home/qukelin/projects/PTOAS/docs/isa/micro-isa/17-simt.md`.

   Basic syntax:

   ```mlir
   pto.keep %value {slot = N : i64} : T
   %value = pto.resume {slot = N : i64} : T
   ```

   Key constraints from the ISA doc:

   - Both ops must appear inside functions marked with `pto.simt_entry`.
   - A `pto.resume` group must be the first non-constant operation group in
     its SIMT entry.
   - A `pto.keep` group must be the final operation group before optional
     `pto.syncthreads` and `func.return`.
   - `slot` is an explicit non-negative `i64` logical slot in `[0, 122]`.
   - The resumed value should use the same type as the value kept in that slot.
2. Run `./build.sh run`.
3. Check build errors from PTOAS/Bisheng if IR is invalid.
4. Check `validation PASS` and elapsed time if the kernel is valid.

The current generated launch uses grid `64` and dynamic UB size `82496` bytes,
matching the TileLang-generated `launch.cpp`.
